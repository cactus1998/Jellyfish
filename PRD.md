# Jellyfish PRD (Discord Online Counter Bot)

## 1. 產品目標
這款機器人的初衷是為了協助伺服器管理者紀錄及監測「最高同時在線人數」。藉由永久儲存最高紀錄，管理者可以分析伺服器的成長情況與活躍時段。

## 2. 功能需求

### 核心功能
- **即時在線監控**：當成員的在線狀態改變 (例如：從離線轉為在線) 時，機器人能迅速捕捉並重新計算。
- **最高紀錄維護**：自動對比當前人數與歷史最高紀錄，若打破紀錄則即時更新。
- **即時人數看板**：允許管理員設定一個語音頻道，即時顯示當前的在線人數。
- **數據持久化**：使用本地檔案系統 (data.json) 儲存數據，確保在機器人重啟、系統當機或斷網後，歷史紀錄依然完整。

### 生日祝賀功能
- **用戶生日存儲**：允許用戶設定自己的生日，並將數據持久化存儲於 `data.json`。
- **自動祝賀機制**：機器人每日定時掃描，若當日有成員生日，則在指定頻道發送美化的祝賀訊息。
- **頻道設定**：提供指令讓管理員設定祝賀訊息發送的頻道。

### 指令功能
- **`/peak` 指令**：提供圖形化或格式化的報表，顯示目前的最高人數紀錄及該紀錄達成的手工時間。
- **`/setchannel` 指令**：允許管理員指定一個語音頻道作為「最高人數看板」，機器人會自動修改該頻道名稱以呈現統計數據。
- **`/setcurrent` 指令**：允許管理員指定一個語音頻道作為「當前人數看板」，機器人會自動修改該頻道名稱以呈現現況。
- **`/online` 指令**：顯示目前的在線人數。
- **`/setbirthday` 指令**：用戶可用此指令設定或更新自己的生日 (格式：MM-DD)。
- **`/admin_setbirthday` 指令**：管理員可用此指令設定或更新其他成員的生日。
- **`/birthday_countdown` 指令**：（管理員專用）顯示指定成員距離下次生日還有多少天。
- **`/birthday_channel` 指令**：允許管理員設定發送生日祝賀訊息的文字頻道。

## 3. 非功能性需求
- **輕量化**：機器人運作所需的記憶體與處理器資源極低。
- **易於部署**：採用 Node.js 架構，僅需填寫 Token 並開啟權限即可運作。
- **高穩定性**：包含初始掃描機制，即使在機器人短暫斷開連線期間，重啟後也能立刻校準現況。

## 4. 技術規格與要求
- **語言/環境**：Node.js (v16.11.0+)。
- **庫**：discord.js (v14+)。
- **權限要求 (Privileged Intents)**：
    - `GuildMembers`：讀取伺服器成員列表。
    - `GuildPresences`：讀取成員狀態 (由離線變更為在線)。
- **資料儲存**：本地 JSON 檔案。

## 5. 未來擴展方向 (可選)
- 視覺化圖表產出。
- 特定時段的在線密度分析。
- 多伺服器分開統計。
- **生日提醒與角色分配**：當天給予生日成員特殊身分組。
